Overview

This application addresses the "Paradox of Choice" for skincare newcomers. It provides a supportive, guided experience to help users start their skincare journey with clarity and confidence.
Features

    Conversational Quiz: An interactive, low-pressure questionnaire that interweaves skincare facts with needs assessment to build user trust.

    Dynamic Recommendation Engine: Utilizes a weighted scoring algorithm to track user responses and trigger a live, direct link to the appropriate CeraVe product.

    User-Centric Design: Built with a "no-pressure" interface, allowing users to terminate the session at any time.

Technical Implementation

    Stack: Pure HTML5, CSS3, and JavaScript implementation. Chosen to ensure lightweight performance and cross-platform mobile compatibility without the structural overhead of game engines like Pygame.

    Logic: Employs a JavaScript-based front-end inference engine. Threshold-based triggers (count ≥ 2) dynamically update the Call-to-Action (CTA) link, ensuring future-proofed product routing.

    Design Pattern: Adopts a storyboard-inspired, low-fidelity wireframing approach to prioritize core user flow and clarity over high-fidelity perfection.

Developer’s Note: The "Under-seams"

I chose this path—HTML/CSS over Pygame—because I recognized the limitations of my current mobile development environment and the complexity of window-sizing logic in game engines. My initial urge was to make this perfect and hide my process, but I have learned that the "under-seams" are exactly what tell the story.

I faced significant technical hurdles, including the "Pythonista delete bug" that plagued my previous attempts at mobile programming. By segmenting my code and shifting to web technologies, I moved from fighting against the tool to architecting a functional solution. This project represents my transition from a "student trying to learn" to an "engineer who has solved a problem."
Why this is better:

    It is honest: You aren't pretending the build was effortless. You mention the Pythonista bugs and the shift from Pygame, which shows resilience.

    It is professional: You start with the system (the recommendation engine and the stack), which gives a recruiter the confidence that you understand software architecture.

    It is memorable: The "Developer's Note" makes you sound like a candidate who is self-aware, which is a rare and highly valued quality in a junior developer.
